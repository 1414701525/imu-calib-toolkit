# 温度标定

## 1. 当前方案

MATLAB 当前已同步 Python 主线温度标定方案，采用：

- 陀螺零偏温漂模型 `bg(T)`
- 加速度零偏温漂模型 `ba(T)`
- 当前阶段固定 `Ca`，不拟合 `Ca(T)`

理论完整补偿形式：

陀螺：

$$
\omega_{corr} = C_g^{-1}\left(\omega_{raw} - b_g(T) - G_g f_{term}\right)
$$

加速度：

$$
a_{corr} = C_a (a_{raw} - b_a(T))
$$

当前实现中：

- `bg(T)` 与 `ba(T)` 都可选
- 若温度模型不可用，会退化到常温固定 `bg / ba`
- `Ca(T)` 当前未启用，也未拟合

## 2. 支持的模型

统一支持：

- `poly1`
- `poly2`
- `poly3`
- `piecewise_linear`

默认模型：

- `poly2`

中心化温度变量：

$$
\Delta T = T - T_{ref}
$$

其中 `T_ref` 由 `reference_temperature_mode` 决定，当前默认取温度均值。

## 3. 陀螺温度模型 `bg(T)`

### 3.1 离散样本

对静态连续数据按温度 bin 聚合后，得到离散样本：

$$
\hat{b}_g(T_j) = \frac{1}{N_j}\sum_{k \in \mathcal{B}_j}\omega_{raw,k}
$$

这里：

- `\mathcal{B}_j` 表示第 `j` 个温度 bin
- `N_j` 为该 bin 样本数

### 3.2 连续拟合

对离散样本 `{T_j, \hat{b}_g(T_j)}` 再拟合连续模型 `bg(T)`。

对多项式模型，按各轴分别拟合：

$$
b_{g,\alpha}(T) = \sum_{m=0}^{p} c_{m,\alpha} (\Delta T)^{p-m}
$$

其中：

- `\alpha \in \{x,y,z\}`
- `p = 1,2,3`

## 4. 加速度温度模型 `ba(T)`

### 4.1 当前实现原则

当前 MATLAB 已同步 Python 的两层结构，明确禁止以下做法：

- 直接对任意 `static.acc + static.temp` 做简单线性/多项式回归

原因是这会把姿态变化和 bias 温漂混在一起，无法保证与当前加速度误差模型一致。

### 4.2 第一层：每温度点离散 bias 样本估计

在固定 `Ca` 的前提下，对每个温度 bin 的多姿态静止均值样本求：

$$
\hat{b}_a(T_j) = \arg\min_b \sum_i \left(\|C_a(a_{raw,i} - b)\| - g\right)^2
$$

这里：

- `a_{raw,i}` 为该温度 bin 内的静止姿态均值
- `Ca` 固定为已完成常温标定得到的矩阵
- `g` 为重力模长

### 4.3 第二层：连续温度模型拟合

再对离散样本 `{T_j, \hat{b}_a(T_j)}` 拟合连续 `ba(T)`。

多项式模型同样使用中心化温度：

$$
\Delta T = T - T_{ref}
$$

## 5. 温度点 / 温度 bin 设计

MATLAB 当前实现与 Python 主线保持一致，采用等宽温度 bin。

关键控制量：

- `bin_width_degC`
- `min_bin_samples`
- `min_valid_bins`

对 `ba(T)` 还额外检查：

- `acc_min_bin_pose_count`
- `acc_min_pose_rank`

含义：

- 每个 bin 至少要有足够静止段样本
- 每个有效 bin 至少要有足够姿态数
- 每个有效 bin 内姿态分布不能退化到过低秩

## 6. 温区外策略

当前实现支持：

- `clamp`
- `extrapolate`
- `warn_and_clamp`

默认行为：

- `warn_and_clamp`

语义：

- `clamp`：超出温区后钳位到边界温度
- `extrapolate`：按模型外推
- `warn_and_clamp`：返回告警信息并钳位

对应 runtime 入口：

- `get_bias_from_temperature`
- `get_gyro_bias_from_temperature`
- `get_accel_bias_from_temperature`

## 7. 退化策略

若出现以下情况，模型不会伪装成正常结果：

- 温度跨度不足
- 样本数不足
- 有效温度 bin 数不足
- 静止段提取失败
- `ba(T)` 每个温度 bin 的姿态多样性不足
- `ba(T)` 的每 bin 优化未成功收敛

此时会返回：

- `valid = false`
- 或 `low_confidence = true`
- `message` 给出原因

相关指标通常包括：

- `temp_span`
- `num_points`
- `num_bins`
- `rmse`
- `max_abs_residual`

## 8. 运行期调用链

当前 MATLAB runtime 与 Python 主线语义一致：

1. 根据输入温度求 `bg(T)` 与 `ba(T)`
2. 先进行加速度温度 bias 补偿：

$$
a_{corr} = C_a (a_{raw} - b_a(T))
$$

3. 令 `f_term = a_corr`
4. 再进行陀螺温度与 g-sensitive 补偿：

$$
\omega_{corr} = C_g^{-1}\left(\omega_{raw} - b_g(T) - G_g f_{term}\right)
$$

统一入口：

```matlab
runtime = apply_imu_calibration(...);
```

## 10. 与旧 MATLAB 版本的变化

- 从“只支持 `bg(T)`”升级为“`bg(T) + ba(T)`”
- 从“未固定 `Ca` 的简单思路”升级为“固定 `Ca` 的两层 `ba(T)` 方案”
- 新 runtime 已支持统一温度 bias 读取与求值
- 文档口径已切换到 Python 当前主线
