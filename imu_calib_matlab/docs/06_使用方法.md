# 06 使用方法

## 1. MATLAB 环境准备

在 MATLAB 中切换到工程目录：

```matlab
cd('F:\IMU\IMU_test\imu_calib_matlab');
addpath(genpath(pwd));
```

若只想从脚本入口运行，通常只需要保证：

- 当前目录在 `imu_calib_matlab/`
- `src/` 已加入路径

## 2. 快速开始

### 2.1 synthetic 示例

```matlab
main_demo
```

该入口会：

1. 生成或加载示例数据
2. 运行完整标定流程
3. 输出 calibration summary

### 2.2 从目录运行

```matlab
task = run_from_csv('data\example_dataset_bundle');
results = task.result.results;
```

### 2.3 从单文件运行

```matlab
task = run_from_csv('data\example_dataset_bundle\static.csv');
```

单文件模式会按可用输入执行部分任务，不会强行要求完整数据集。

## 3. 主要任务入口

### 3.1 完整流程

```matlab
task = run_full_calibration(inputData);
```

可接受：

- 目录路径
- manifest 目录
- 单个 CSV 文件
- 内存结构体

### 3.2 加速度计标定

```matlab
task = run_acc_calibration(data);
Ca = task.result.Ca;
ba = task.result.ba;
```

输入可为：

- `data.accPoses`
- 或可自动提取静止段的 `data.static`

### 3.3 陀螺零偏

```matlab
task = run_gyro_bias(data);
bg = task.result.bg;
```

### 3.4 陀螺矩阵 `Cg`

```matlab
task = run_gyro_calibration(data, 'bg', bg);
Cg = task.result.Cg;
```

### 3.5 `Gg` 拟合

```matlab
task = run_gsens_fit(data, 'bg', bg, 'Cg', Cg);
Gg = task.result.Gg;
```

### 3.6 温度模型

仅陀螺：

```matlab
task = run_temperature_fit(data, 'target', 'gyro');
bgModel = task.result.bgModel;
```

陀螺 + 加速度：

```matlab
task = run_temperature_fit(data, ...
    'target', 'both', ...
    'Ca', Ca, ...
    'ba', ba);
bgModel = task.result.bgModel;
baModel = task.result.baModel;
```

## 4. 运行期补偿

统一入口：

```matlab
runtime = apply_imu_calibration(gyroRaw, accRaw, ...
    'temperature', temp, ...
    'Cg', Cg, ...
    'Ca', Ca, ...
    'Gg', Gg, ...
    'bg', bg, ...
    'ba', ba, ...
    'bgModel', bgModel, ...
    'baModel', baModel);
```

核心输出：

- `runtime.gyroCalibrated`
- `runtime.accCalibrated`
- `runtime.bgT`
- `runtime.baT`
- `runtime.info.temperature`

## 5. 结果组织

完整流程通常返回：

```matlab
task.result.results
```

主要字段：

- `results.calib`
- `results.analysis`
- `results.validation`
- `results.compat`

其中 `results.calib.temp` 下包含：

- `bgModel`
- `baModel`
- `reference_temperature`
- `temperature_range`

## 6. 保存结果

```matlab
save_calibration_results(results, 'output');
```

当前 MATLAB 输出：

- `calibration_results.mat`
- `calibration_summary.json`
- `calibration_arrays.mat`

说明：

- 语义已尽量与 Python 主线统一
- 存储介质仍是 MATLAB 友好的 `MAT + JSON`
- 尚未完全复刻 Python 的 `pkl / npz` 物理文件格式

## 7. 如何验证结果

### 7.1 结构检查

重点检查：

- `Ca / Cg / Gg` 是否存在
- `bgModel / baModel` 是否带 `valid / low_confidence / message`
- `temperature_range / reference_temperature / extrapolation_mode` 是否存在

### 7.2 数值检查

建议比较：

- 加速度静止段补偿前后模长是否更接近 `g`
- 陀螺静态补偿后均值是否更接近零
- 相同温度输入下，`bg(T)` / `ba(T)` 求值是否连续稳定

### 7.3 与 Python 对比

建议在同一份输入数据上比较：

- `Ca`
- `Cg`
- `Gg`
- `bg(T)` 求值
- `ba(T)` 求值
- runtime 补偿后的主要数值

结果允许存在语言和优化器带来的小数值误差，但语义与口径应一致。

## 8. 常见失败场景

### 加速度标定失败

优先检查：

- 姿态数是否足够
- 是否真的为静止均值
- 多姿态覆盖是否过于单一

### `ba(T)` 无法拟合

优先检查：

- 是否已有固定 `Ca`
- `static` 中是否能提取多个静止段
- 每个温度 bin 是否有足够姿态覆盖
- 温度跨度是否过小

### 只得到部分结果

这是允许的。`run_full_calibration` 在输入不完整时会自动退化为 partial 模式，并在 `warnings` 与 `validation.summary` 中给出可用模块说明。

- MATLAB 当前已同步主算法链路和运行期接口
- 自动化验证仍依赖本机 MATLAB 环境
