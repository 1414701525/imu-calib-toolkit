# 05 输入文件与数据采集要求

## 1. 支持的输入形式

MATLAB 当前支持以下输入方式：

- 数据目录：包含标准 CSV 文件集合
- manifest bundle 目录：包含 `dataset_manifest.json`
- 单文件输入：
  - `static.csv`
  - `acc_poses.csv`
  - `gyro_runs.csv`
  - `gsens_runs.csv`
- 内存结构体输入：字段语义与 Python 主线保持一致

统一入口：

```matlab
data = load_csv_data(inputPath);
task = run_full_calibration(inputPath);
```

## 2. 标准文件集合

标准目录下推荐文件：

- `static.csv`
- `acc_poses.csv`
- `gyro_runs.csv`
- `gsens_runs.csv`
- `dataset_manifest.json` 可选

不同模块的最小依赖：

| 模块 | 最小输入 |
| --- | --- |
| 陀螺静态零偏 `bg` | `static.csv` |
| 噪声统计 / Allan | `static.csv` |
| 加速度静止多姿态标定 | `acc_poses.csv` 或可提取静止段的 `static.csv` |
| 陀螺 `Cg` 标定 | `gyro_runs.csv` + 可用 `bg` |
| `Gg` 标定 | `gsens_runs.csv` + 可用 `bg` + `Cg` |
| `bg(T)` | `static.csv` 中的 `gyro + temp` |
| `ba(T)` | `static.csv` 中的 `acc + temp`，且必须已有固定 `Ca` |

## 3. `static.csv`

必需列：

- `t`
- `gx`
- `gy`
- `gz`
- `ax`
- `ay`
- `az`

可选列：

- `temp`

字段要求：

- `t` 单调严格递增
- `gx/gy/gz` 单位应与整个工程一致，当前默认按 `rad/s` 理解
- `ax/ay/az` 单位应为 `m/s^2`
- `temp` 建议为摄氏度

用途：

- 静态零偏估计
- 静态噪声统计
- Allan 分析
- 静止段提取
- `bg(T)` 拟合
- `ba(T)` 的静止段/温度 bin 两层估计

## 4. `acc_poses.csv`

新主线必需列：

- `pose_name`
- `acc_x`
- `acc_y`
- `acc_z`

兼容列：

- `ax/ay/az`
- legacy `ref_x/ref_y/ref_z`

说明：

- 当前 MATLAB 已同步 Python 新方案，不再要求逐姿态参考加速度向量
- `ref_x/ref_y/ref_z` 若存在，必须三列同时存在
- legacy 参考列只用于初始化兼容，不再作为主目标函数约束

当前加速度标定模型：

$$
a_{corr} = C_a (a_{raw} - b_a)
$$

优化目标为多姿态静止重力模长约束：

$$
\min \sum_i \left(\|C_a(a_{raw,i}-b_a)\| - g\right)^2
$$

采集建议：

- `scale_misalignment` 模式建议至少 9 个姿态
- 每个姿态都应充分静止后再取均值
- 姿态分布尽量覆盖三轴正负方向附近

## 5. `gyro_runs.csv`

必需列：

- `run_id`
- `axis`
- `dir`
- `t`
- `gx`
- `gy`
- `gz`
- `idx_ss`
- `theta_ref_x`
- `theta_ref_y`
- `theta_ref_z`

字段要求：

- 同一 `run_id` 内，`theta_ref_*` 必须保持常数
- `axis` 只能为 `x/y/z`
- `dir` 只能为 `+1/-1`
- `idx_ss` 只能为 `0/1`
- `t` 必须严格递增

## 6. `gsens_runs.csv`

必需列：

- `run_id`
- `t`
- `gx`
- `gy`
- `gz`
- `acc_ref_x`
- `acc_ref_y`
- `acc_ref_z`

可选列：

- `omega_ref_x`
- `omega_ref_y`
- `omega_ref_z`

说明：

- 若提供 `omega_ref_*`，则必须三列同时存在
- `acc_ref_*` 表示拟合 `Gg` 时使用的参考比力输入

## 7. 列名映射

MATLAB loader 已同步 Python 的常用别名归一化，支持例如：

- `time -> t`
- `gyro_x / gyrox -> gx`
- `gyro_y / gyroy -> gy`
- `gyro_z / gyroz -> gz`
- `acc_x / accel_x / accx -> ax`
- `acc_y / accel_y / accy -> ay`
- `acc_z / accel_z / accz -> az`

对 `acc_poses.csv`：

- `ax/ay/az` 会自动映射到 `acc_x/acc_y/acc_z`

## 8. 温度模型输入要求

### 8.1 `bg(T)`

至少需要：

- 连续静态陀螺数据
- 对应温度列 `temp`
- 足够温度跨度

当前实现流程：

1. 按温度等宽 bin 聚合
2. 每个 bin 求均值，得到离散 `\hat{b}_g(T_j)`
3. 再拟合连续模型 `bg(T)`

### 8.2 `ba(T)`

至少需要：

- 连续静态加速度数据
- 对应温度列 `temp`
- 已有固定 `Ca`
- 能从连续数据中提取多个静止段

当前 MATLAB 已同步 Python 两层方案，不允许把任意 `static.acc + static.temp` 直接做简单回归。

实现流程：

1. 从 `static` 中提取静止段
2. 对每个静止段求 `acc_mean` 与 `temp_mean`
3. 按温度 bin 分组
4. 在固定 `Ca` 下，对每个 bin 内多姿态样本解：

$$
\hat{b}_a(T_j) = \arg\min_b \sum_i \left(\|C_a(a_{raw,i}-b)\| - g\right)^2
$$

5. 对离散样本 `{T_j, \hat{b}_a(T_j)}` 拟合连续 `ba(T)`

额外要求：

- 每个温度 bin 至少有足够样本
- 每个温度 bin 至少覆盖足够多姿态
- 每个温度 bin 的姿态分布需要满足最小秩要求

若不满足，则模型会返回：

- `valid = false`
- 或 `low_confidence = true`
- 同时附带 `message`

## 9. 温区外与退化策略

当前 MATLAB 与 Python 一致支持：

- `poly1`
- `poly2`
- `poly3`
- `piecewise_linear`

默认：

- `poly2`

温度中心化：

$$
\Delta T = T - T_{ref}
$$

温区外策略：

- `clamp`
- `extrapolate`
- `warn_and_clamp`

若样本数不足、温度跨度不足、有效 bin 不足或姿态可观测性不足，模型会显式退化，而不是静默输出高置信度结果。

## 10. 采集建议

- `acc_poses.csv` 建议采集 9 个以上静止姿态
- 温度标定建议在多个温度平台重复多姿态静止采集
- 每个温度点尽量覆盖 6 个以上有效静止姿态
- 每段静止持续时间要足够，保证均值稳定
- 温度变化过程中应避免把明显动态段直接混入静止均值

## 11. 与旧版 MATLAB 的变化

- 加速度标定主输入已从“`acc_mean + a_ref`”切换为“多姿态静止均值”
- `ref_x/ref_y/ref_z` 不再是必填列
- `ba(T)` 已从“仅陀螺温度模型”阶段升级为“陀螺 + 加速度 bias 温度模型”
- 运行期读取参数时，优先使用新结构中的 `bgModel / baModel`
