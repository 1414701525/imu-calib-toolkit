# v0.1.0 - Initial Public Release

**Release date**: 2026-06-11

## Overview

IMU Calibration Toolkit is a dual-language (Python + MATLAB) toolkit for IMU sensor calibration and runtime compensation. It provides a complete pipeline covering accelerometer calibration, gyroscope analysis, cross-axis compensation, temperature modeling, and production-ready runtime compensation.

This is the initial public release, prepared for open-source contribution and community use.

## Features

### Calibration Capabilities

- **Accelerometer calibration** — multi-pose static calibration with gravity magnitude constraint
- **Gyroscope calibration** — bias estimation, noise statistics, Allan deviation analysis
- **Cross-axis compensation** — Cg matrix fitting with Kg/Mg decomposition
- **G-sensitivity** — Gg matrix fitting for gravity-dependent gyro bias
- **Temperature compensation** — bg(T) and ba(T) temperature-dependent bias models
- **Runtime compensation** — single-call `apply_imu_calibration()` for production use

### Python Implementation

- Full calibration pipeline via CLI (`python -m imu_calib`)
- CSV/manifest/single-file/in-memory input support
- Partial dataset execution (static-only, gyro-only, etc.)
- Validation: static segments, pose errors, angular increment residuals
- Deterministic synthetic demo data (RNG seed = 42)

### MATLAB Implementation

- Synchronized implementation sharing same algorithm semantics
- `main_demo.m` and `run_from_csv.m` entry points
- Python ↔ MATLAB mapping table for cross-reference

### Documentation

- Error model and parameter definitions
- Calibration scheme descriptions
- API references and usage guides
- Developer notes and migration guides

### Testing

- Python pytest test suite
- MATLAB test scripts
- Synthetic dataset for reproducible validation

## Known Limitations

- **Synthetic data only** — no real hardware datasets included; users must provide their own data
- **No CI/CD pipeline** — tests must be run manually
- **No PyPI package** — installation requires cloning the repository
- **MATLAB path setup** — requires manual `addpath(genpath(...))` configuration
- **Limited validation plots** — visualization could be expanded for diagnostics
