## 2. 当前主线的设计原则

### 2.1 加速度参数化不能完全自由

因为仅靠模长约束：

$$
\|C(a_{raw}-b)\| = g
$$

若把 $C$ 当成完全自由 3x3 矩阵，会出现旋转不唯一问题。  
因此当前主线显式采用：

$$
C = S M
$$

### 2.2 partial loader 不能退化

当前 `load_csv_data.py` 支持：

- 目录
- manifest
- 单文件
- 部分加载

不要把它改回“缺一文件全失败”的旧模式。

### 2.3 任务层负责依赖检查

依赖检查应放在：

- `run_acc_calibration.py`
- `run_gyro_calibration.py`
- `run_gsens_fit.py`

不要塞回 loader。

## 3. 最容易出错的地方

### 3.1 新旧 `Ca` 物理意义混淆

这是当前最大的风险点之一。  
当前 `Ca` 是补偿矩阵，不再是旧正向模型中的测量矩阵。

### 3.2 legacy `a_ref` 被误当成主输入

当前 `a_ref` 只是兼容字段，不能再在新功能里被当成必需输入。

### 3.3 `f_term` 定义漂移

当前运行时 `f_term = acc_calibrated`。  
若调整加速度补偿模型，也要同步检查：

- `apply_imu_calibration.py`
- `fit_gyro_g_sensitivity.py`

## 4. 如果后续要继续增强

### 4.1 想增加更强初值策略

优先改：

- `_build_initial_guess(...)`
- `_legacy_reference_initial_guess(...)`

### 4.2 想增加鲁棒估计

优先改：

- `fit_acc_multi_pose.py` 中 `least_squares(...)` 的 `loss`
- 或静止段筛选逻辑

### 4.3 想同步 MATLAB

需要重点迁移：

- `fit_acc_multi_pose.m`
- `run_acc_calibration.m`
- `apply_imu_calibration.m`
- `load_csv_data.m`
- 文档
