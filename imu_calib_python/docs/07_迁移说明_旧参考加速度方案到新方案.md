# 07 迁移说明：旧参考加速度方案到新方案

## 1. 迁移背景

旧方案要求在 `acc_poses.csv` 中为每个静止姿态提供参考加速度向量：

- `ref_x`
- `ref_y`
- `ref_z`

并按旧模型：

$$
a_m = C_a a_{ref} + b_a
$$

进行线性最小二乘拟合。

当前 Python 主线已经切换为新方案：

$$
a_{corr} = C(a_{raw} - b)
$$

并使用静止约束：

$$
\|a_{corr}\| = g
$$

## 2. 原方案依赖什么

旧方案依赖：

- 每个姿态的理论重力方向
- 手工给出 `ref_x/ref_y/ref_z`

## 3. 新方案替代了什么

新方案不再要求：

- 逐姿态参考方向
- `ref_x/ref_y/ref_z`
- 手工姿态方向映射

## 4. 输入文件变化

### 4.1 旧格式

```csv
pose_name,acc_x,acc_y,acc_z,ref_x,ref_y,ref_z
```

### 4.2 新格式

```csv
pose_name,acc_x,acc_y,acc_z
```

## 5. 输出参数变化

旧方案中的 `Ca` 表示正向测量矩阵；  
当前 Python 主线中的 `Ca` 直接表示补偿矩阵：

$$
a_{corr} = C_a(a_{raw} - b_a)
$$

因此旧结果文件不能直接当成新结果文件混用。

## 6. 补偿链路变化

旧方案通常使用：

$$
\hat a_{ref} = C_a^{-1}(a_m - b_a)
$$

新方案改为：

$$
a_{corr} = C_a(a_{raw} - b_a)
$$

## 7. 当前兼容策略

Python 主线仍允许读取旧 `acc_poses.csv` 中的：

- `ref_x`
- `ref_y`
- `ref_z`

但这些字段仅作为 legacy 初始化辅助，不再参与新目标函数。

## 8. MATLAB / Python 的迁移状态

- Python：已迁移
- MATLAB：当前仍保留旧方案，尚未同步

## 9. 建议的迁移步骤

1. 停止在新数据采集中生成 `ref_x/ref_y/ref_z`
2. 预处理脚本只输出静止姿态均值
3. 用 Python 主线重新标定
4. 使用新的 `Ca / ba` 进行补偿
