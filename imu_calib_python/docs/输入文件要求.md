# 输入文件要求

本文聚焦温度标定和运行期补偿最关心的输入要求。完整输入规范仍以 [05_输入文件与数据采集要求.md](05_输入文件与数据采集要求.md) 为主。

## 1. `bg(T)` 需要什么

至少需要：

- `static.gyro`
- `static.temp`

推荐：

- `static.t`

可选：

- `segment_id`（当前标准 loader 未要求）

## 2. `ba(T)` 需要什么

至少需要：

- `static.acc`
- `static.temp`
- 固定 `Ca`

更严格地说，`ba(T)` 需要：

- 连续原始静止 / 姿态切换数据
- 能从中提取多个静止段
- 每个温度点附近有足够多的多姿态静止段

## 3. 不再需要什么

当前温度与加速度主线中，不再要求：

- `acc_poses.csv` 中逐姿态参考向量 `ref_x/ref_y/ref_z`
- 手工给每个温度点标参考方向

## 4. 不能怎样做

当前工程明确不支持以下做法作为 `ba(T)` 主方法：

- 只拿单姿态静态 `acc + temp` 直接回归 `ba(T)`
- 不提供固定 `Ca` 就尝试拟合 `ba(T)`

## 5. 采集建议

为提高 `ba(T)` 可观测性，建议：

- 每个温度点至少采集 6 个以上静止姿态
- 姿态尽量覆盖 `±X / ±Y / ±Z`
- 每个静止姿态保持足够时长，确保能提取稳定均值
- 升温 / 降温过程中尽量覆盖更宽温区

## 6. 当前代码中的主要检查项

温度模块会检查：

- `temp_span`
- `num_samples`
- `num_bins`
- `acc_min_bin_pose_count`
- `acc_min_pose_rank`

不足时会返回：

- `valid = False`
或
- `low_confidence = True`

并给出明确 `message`。
